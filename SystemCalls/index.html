System calls

A system call is a procedure that provides the interface between a process and the operating system. It is the way by which a computer program requests a service from the kernel of the operating system.

Process Control

These system calls deal with processes such as process creation, process termination etc. The Windows System calls under this are Createprocess(),Exitprocess(). The Linux System calls under this are mentioned below

  1. fork()
    1. A new process is created by the fork() system call
    2. A new process may be created with fork() without a new program being run-the new subprocess simply continues to execute exactly the same program that the first (parent) process was running.
    3. It is one of the most widely used system calls under process management.
  2. exit()
    1. The exit() system call is used by a program to terminate its execution.
    2. The operating system reclaims resources that were used by the process after the exit() system call.
  3. exec()
    1. A new program will start executing after a call to exec().
    2. Running a new program does not require that a new process be created first: any process may call exec() at any time. The currently running program is immediately terminated, and the new program starts executing in the context of the existing process.
  4. wait()
    1. wait for single object. A call to wait() blocks the calling process until one of its child processes exits or a signal is received. After child process terminates, parent continues its execution after wait system call.
  5. clear()

      6. clearing out the terminal. clear is a standard Unix computer operating system command that is used to clear the terminal screen.

File Structure

  1. open()
    1. Opens a file and returns a file descriptor.
    2. Arguments: Filename to open, mode (read-only, write-only, read-write), permissions (if creating a new file).
    3. Return type: A file descriptor, or -1 if an error occurs.
  2. read()
    1. Followed by the file name to display its contents. This system call is used to read data from a file. 4
    2. It takes three arguments, the first is the file descriptor, the second is a buffer where the data will be stored, and the third is the number of bytes to read.
  3. write()
    1. This system call is used to write data to a file.
    2. It takes three arguments, the first is the file descriptor, the second is a buffer containing the data to be written, and the third is the number of bytes to be written.
  4. close()
    1. This system call is used to close a file.
    2. It takes a file descriptor as an argument and releases the resources associated with the file descriptor
